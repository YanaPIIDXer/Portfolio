# ブランチの扱い方

## 各ブランチの概要

一般的なgit flow。  

|ブランチ名|概要|派生元ブランチ|push可能？|
|---|---|---|---|
|master|マスタブランチ||×|
|develop|開発のメインブランチ|master|×|
|feature/XXX|機能実装用ブランチ|develop|○|
|bugfix/XXX|不具合修正用ブランチ|develop|○|
|hotfix/XXX|緊急対応用ブランチ|master|○|

※リポジトリをサービス毎に分割する方式を取ったので、このリポジトリのブランチ構成は変更になる可能性あり。
